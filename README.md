# ⚗️ Фабрика гипотез

Инструмент на базе ИИ для **автоматической генерации и ранжирования научно-исследовательских
гипотез** в материаловедческих и технологических НИОКР-проектах.

На вход: **целевой показатель (KPI)** + **база знаний** (литература, отчёты, эксперименты).
На выход: **структурированный ранжированный список проверяемых гипотез** с обоснованием,
оценкой новизны, рисков, реализуемости и потенциальной ценности — и с планом эксперимента для проверки.

> Не «чёрный ящик»: каждая рекомендация прозрачна — видно, на какие источники она опирается,
> как посчитан итоговый ранг и как эксперт может его скорректировать.

---

## Ключевые идеи

| Принцип | Как реализовано |
|---|---|
| **Интерпретируемость** | Модель оценивает гипотезу по 4 названным осям (новизна / ценность / реализуемость / риск) **с письменным обоснованием каждой**. Итоговый ранг — **прозрачная взвешенная формула**, а не «оценка от ИИ». |
| **Экспертная корректировка** | Любую оценку можно переопределить вручную, изменить статус (принять/отклонить/на проверке), добавить заметки, отредактировать текст гипотезы. |
| **Снижение субъективности** | Гипотезы строятся из общей базы знаний по воспроизводимому пайплайну, а не из опыта одного сотрудника. Все запуски логируются (аудит). |
| **Качественный RAG** | Двухэтапный поиск: дешёвый **TF-IDF** отбирает кандидатов, точный **реранкер** (cohere/rerank-4-fast) их упорядочивает. Кросс-энкодер находит релевантные источники, которые мешок слов пропускает (в тестах: пассаж с TF-IDF≈0.0 получал реранк 0.75). |
| **Веса — модель И эксперт** | Веса ранжирования может предложить **модель** (анализирует стадию/ограничения проекта и обосновывает расстановку) или задать **эксперт** ползунками. |
| **Экономия ресурсов** | TF-IDF локальный; реранкер видит только N кандидатов (не весь корпус); DeepSeek компактная (~$0.001–0.002 за пакет из 5 гипотез, реранк ~$0.002). Пере-ранжирование по весам — мгновенно на клиенте, без повторного вызова модели. |

## Как это работает (ML-пайплайн)

```
 Тема / KPI / Ограничения
        │
        ▼
 ┌─────────────────────┐   умный парсер: OpenAlex + Crossref + Semantic Scholar
 │ Сбор базы знаний     │──── (+ Materials Project по ключу). Топик → импорт
 │ (внешние + локальные)│     релевантных статей/данных в базу проекта
 └─────────┬───────────┘
           ▼
 ┌─────────────────────┐   этап 1: chunking → TF-IDF отбирает N кандидатов
 │  RAG (2 этапа)       │   этап 2: реранкер cohere/rerank-4-fast упорядочивает
 │  retrieve → rerank   │──── по релевантности (кросс-энкодер). Диверсификация
 └─────────┬───────────┘     по источникам. Оба этапа логируются (прозрачность)
           ▼
 ┌─────────────────────┐   веса: предлагает МОДЕЛЬ (под стадию проекта) ИЛИ эксперт
 │ Веса ранжирования    │──── novelty / value / feasibility / risk
 └─────────┬───────────┘
           ▼
 ┌─────────────────────┐   промпт с пронумерованными источниками [S1..Sk]
 │ Генерация (DeepSeek) │──── гипотезы: формулировка, связь с целью, механизм,
 └─────────┬───────────┘     план проверки, ссылки на источники, 4 оценки
           ▼
 ┌─────────────────────┐   composite = Σ(wᵢ·sᵢ)/Σ(wᵢ),  вклад риска = (100−risk)
 │ Прозрачное ранжир-е  │──── веса задаёт эксперт (ползунки, live re-rank)
 └─────────┬───────────┘
           ▼
   Ранжированный список  →  экспертная корректировка  →  экспорт (MD / CSV / JSON)
```

## Технологический стек

- **Frontend** — React 18 + Vite (nginx в проде). Без тяжёлых UI-библиотек.
- **Backend** — Python **Flask** + SQLAlchemy, gunicorn.
- **БД** — **PostgreSQL** 16.
- **LLM** — **DeepSeek** через **OpenRouter** (OpenAI-совместимый API).
- **RAG** — двухэтапный: scikit-learn **TF-IDF** (локальный отбор) → **cohere/rerank-4-fast** (реранкер через OpenRouter).
- **Внешние источники** — OpenAlex, Crossref, Semantic Scholar (keyless), Materials Project (по ключу).
- **Оркестрация** — **docker-compose**.

## 🚀 Быстрый старт

Нужен только Docker с плагином Compose.

```bash
# 1. Ключ OpenRouter уже прописан в .env (выдан заказчиком).
#    При необходимости отредактируйте .env (OPENAI_MODEL и т.д.)

# 2. Собрать и запустить весь стек
docker compose up --build
```

Откройте **http://localhost:3000**

- Frontend: http://localhost:3000
- Backend API: http://localhost:5000/api/health
- PostgreSQL: localhost:5432 (`hypo` / `hypo_secret`)

При первом запуске автоматически создаётся **демо-проект** (коррозионностойкий Ni-сплав)
с базой знаний из 8 источников — можно сразу нажать «⚡ Сгенерировать гипотезы».

> Генерация занимает ~1–2 минуты: DeepSeek — reasoning-модель и рассуждает над источниками.
> Это осознанный размен «немного времени ↔ качество и обоснованность гипотез».

> Модель легко заменить на любую доступную в РФ через OpenRouter (например, другую компактную
> DeepSeek) — просто поменяйте `OPENAI_MODEL`. Меньше модель при сохранении точности — дешевле.

## Формула ранжирования (полностью прозрачна)

```
composite = ( wн·Новизна + wц·Ценность + wр·Реализуемость + wриск·(100 − Риск) ) / (wн + wц + wр + wриск)
```

Веса `w` эксперт задаёт ползунками. Риск входит инвертированно (больше риск → ниже ранг).
Пере-ранжирование при смене весов происходит мгновенно на клиенте — модель повторно не вызывается.
